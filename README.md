# Colosseum

Colosseum is a multi-LLM chat arena. One prompt is sent to OpenAI, Claude, and Gemini in parallel, each model gives an initial answer, then each model critiques the other two. Finally, all three responses and critiques are evaluated and compared, and a summary with a winner is generated. Results are displayed side by side in a React UI backed by FastAPI and LangGraph. Chat summaries are persisted to PostgreSQL.

## Stack

- **Frontend:** React + TypeScript + Vite
- **Backend:** LangGraph, LangSmith, DeepEval, FastAPI, Pydantic, PostgreSQL (with SQLAlchemy async ORM)
- **AI Providers:** OpenAI, Anthropic Claude, Google Gemini
- **Database:** PostgreSQL 16 (with asyncpg async driver)

## How it works

**Phase 1 вЂ” Initial responses (parallel)**  
All three models answer the user's question simultaneously.

**Phase 2 вЂ” Cross-critique (parallel)**  
Each model receives the other two models' responses and critically evaluates them. All critiques run in parallel.

**Phase 3 вЂ” Evaluation & Scoring**  
DeepEval judges each response and critique on:
- **Correctness** вЂ” factual accuracy
- **Completeness** вЂ” coverage of the topic
- **Reasoning** вЂ” logical structure
- **Clarity** вЂ” readability and explanation

Evaluations are run by all three models (each acts as a judge for the others).

**Phase 4 вЂ” Summary & Winner**  
A pairwise comparison is performed between all three models, and a winner is determined by majority vote.  
A summary is generated by OpenAI summarizing the top 3 responses in their respective languages.

Chat session summaries are automatically persisted to PostgreSQL.

All phases are orchestrated by a LangGraph `StateGraph`, and the backend streams results via Server-Sent Events (SSE) to the frontend.

## 1. Running with Docker Compose (recommended)

> **Prerequisites:** Docker Desktop must be installed and **running** before executing any `docker` command.  
> On **Windows** and **macOS**, Docker Desktop does not start automatically вЂ” open it from the Start menu / Applications and wait until the whale icon in the system tray shows "Docker Desktop is running".  
> On **Linux**, the Docker daemon starts automatically as a system service, so no manual step is needed.

**Step 1 вЂ” Create `backend/.env` with your API keys:**

```bash
cp backend/.env.example backend/.env
# then edit backend/.env and fill in your keys
```

```env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=...
LANGSMITH_API_KEY=...  # optional, for LangSmith tracing
```

**Step 2 вЂ” Runtime configuration (config/config.yaml):**

Non-secret settings are configured in `config/config.yaml`:

```yaml
models:
  openai: gpt-4.1-mini
  anthropic: claude-haiku-4-5-20251001
  gemini: gemini-2.5-flash

gemini:
  timeout_seconds: 30
  max_retries: 3
  retry_backoff_seconds: 1.0

deepeval:
  per_attempt_timeout_seconds_override: 180

langsmith:
  tracing: true
  project: colosseum
  endpoint: https://api.smith.langchain.com

database:
  url: postgresql+asyncpg://colosseum:colosseum@localhost:5432/colosseum

frontend:
  api_base: http://localhost:8000
```

**Step 3 вЂ” Build and start all services:**

```bash
docker compose up --build
```

Services will start:
- **PostgreSQL:** `postgres:16-alpine` on port 5432
- **Backend:** FastAPI on port 8000
  - Health check: http://localhost:8000/health
  - Streaming chat: POST http://localhost:8000/chat/stream
- **Frontend:** Vite on port 5173

The backend image is built with `uv` вЂ” no `requirements.txt` needed, dependencies come from `pyproject.toml`.

If containers are already built, restart without rebuilding:

```bash
docker compose up
```

## 2. Manual setup вЂ” Backend

**Prerequisites:**
- Python 3.10+
- PostgreSQL 13+ running locally
- API keys from OpenAI, Anthropic, Google

**Step 1 вЂ” Install uv (if not already installed):**

```bash
pip install uv
```

**Step 2 вЂ” Set up backend:**

```bash
cd backend
uv sync
cp .env.example .env
# Edit .env and fill in your API keys
```

**Step 3 вЂ” Create local PostgreSQL database:**

```bash
# On macOS with Homebrew:
brew install postgresql@16
brew services start postgresql@16
createdb -U postgres colosseum

# Or use Docker:
docker run --rm -d \
  -e POSTGRES_USER=colosseum \
  -e POSTGRES_PASSWORD=colosseum \
  -e POSTGRES_DB=colosseum \
  -p 5432:5432 \
  postgres:16-alpine
```

**Step 4 вЂ” Update backend/.env with your database URL:**

```env
DATABASE_URL=postgresql+asyncpg://colosseum:colosseum@localhost:5432/colosseum
```

**Step 5 вЂ” Start the backend:**

```bash
uv run uvicorn main:app --reload
```

Backend listens on http://localhost:8000.  
The lifespan startup handler automatically creates tables if they don't exist.

## 4. API

### POST `/chat/stream`

Streams chat results via Server-Sent Events (SSE).

**Request body:**

```json
{ "message": "What is the difference between RAG and fine-tuning?" }
```

**Response (streaming events):**

Each event is a JSON object with:
- `type`: `"node_update"`, `"done"`, or `"error"`
- `node`: name of the graph node that just completed
- `responses`, `critiques`, `evaluations`, `summary`: arrays/objects as they arrive

**Final event (`type: "done`):**

```json
{
  "type": "done",
  "responses": [
    { "provider": "openai", "model": "gpt-4.1-mini", "content": "...", "error": null },
    { "provider": "anthropic", "model": "claude-haiku-4-5-20251001", "content": "...", "error": null },
    { "provider": "google", "model": "gemini-2.5-flash", "content": "...", "error": null }
  ],
  "critiques": [
    { "provider": "openai", "critiqued_providers": ["anthropic", "google"], "content": "...", "error": null },
    ...
  ],
  "evaluations": [
    { "provider": "openai", "component": "response", "scores": { "correctness": 8.5, "completeness": 9.0, ... }, "judge_model": "gpt-4.1-mini", "contestants": ["anthropic", "google"], "winner": "openai", "reason": "..." },
    ...
  ],
  "summary": {
    "summary": "OpenAI: ...\nClaude: ...\nGemini: ...",
    "winner": "openai",
    "is_tie": false,
    "error": null
  }
}
```

### POST `/chat`

Non-streaming version (returns full response once complete).

**Request body:**

```json
{ "message": "..." }
```

**Response:**

Same structure as the `done` event from `/chat/stream`.

### GET `/results`

Returns cached responses from the most recent `/chat` request.

### GET `/health`

Returns `{ "status": "ok" }`.

## Database

Chat summaries are persisted to PostgreSQL automatically. The `ChatSession` table stores:

```sql
CREATE TABLE chat_sessions (
  id UUID PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE,
  user_message TEXT NOT NULL,
  summary TEXT,
  winner VARCHAR(100),
  is_tie BOOLEAN DEFAULT FALSE
);
```

**Without Docker:**  
Set `DATABASE_URL` in `backend/.env`. If not set, saving is silently skipped (app still works).

**With Docker:**  
PostgreSQL is automatically created and injected via `docker-compose.yml`.

## Notes

- Streaming via SSE provides real-time feedback as models respond.
- DeepEval evaluation timeout is configurable in `config/config.yaml` (`deepeval.per_attempt_timeout_seconds_override`).
- Gemini timeout is configurable in `config/config.yaml` (`gemini.timeout_seconds`).
- Language matching: critique and summary prompts respect the original question's language.
- Frontend auto-scrolls milestone-based (new turn, responses complete, critiques complete).
- Chat history is not persisted вЂ” each request is independent. Persistence feature for multi-turn conversations is planned.
